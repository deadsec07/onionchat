CLI-only. Tor-based. No accounts.

Private chat for people who want less surface area.

onionchat is a minimal Rust CLI for peer-to-peer text chat over Tor onion services. No phone number. No email. No central server. Linux, macOS, Windows, and Android via Termux.

View Source Download Releases

What It Does

  • Signed invite export and import for direct contact exchange
  • Direct peer-to-peer messaging over Tor onion services
  • Persistent local identity and peer address book
  • Single-message send, direct terminal chat, and invite-only groups
  • Tor control-port integration with SAFECOOKIE support
  • Runs on Android through Termux without needing a native app rewrite

What It Does Not Do

  • No file transfer
  • No public discovery or username lookup
  • No cloud backend
  • No telemetry, analytics, or tracking

Quick Start

Run a local Tor daemon with a SOCKS port and control port, then:

cargo build --release
./target/release/onionchat init
./target/release/onionchat invite export --name alice --output alice-invite.json
./target/release/onionchat identity show
./target/release/onionchat listen

From another peer:

./target/release/onionchat invite import alice-invite.json
./target/release/onionchat chat <peer_onion>

Android support is currently through Termux with a local Tor process.

Threat Model

onionchat reduces exposure by avoiding central infrastructure and routing transport over Tor. It does not guarantee users are untrackable.

Timing analysis, endpoint compromise, OS compromise, Tor misconfiguration, and user mistakes can still deanonymize users.

Why It Stays Small

The project is intentionally narrow so the code stays readable, auditable, and practical to distribute as a standalone binary.

The goal is a realistic MVP, not a kitchen-sink messenger.
